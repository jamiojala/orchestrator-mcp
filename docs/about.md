# About

`orchestrator-mcp` exists because modern AI coding workflows are already multi-agent and multi-provider.

Most teams now mix:

- Codex for repo-native implementation
- Claude Code for planning and review
- Kimi Code for IDE-first iteration
- Ollama for local or subscription-based open-model capacity
- Gemini and NVIDIA-backed models for multimodal review and reasoning-heavy sidecars

This project gives those workflows:

- one local orchestration layer
- one safety model
- one routing surface
- one reusable skill marketplace
